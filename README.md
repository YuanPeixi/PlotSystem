# Plot System
----
## 1.项目目的
最终目的是实现一个多智能体、可分支的结局剧情推演系统

## 2.相关设计

### 智能体设计
- 导演智能体：核心控制，负责创建场景，统筹整个剧情
- 角色智能体：负责沉浸的扮演角色（采用类似SillyTavern的设计来进一步增强人设）
- 环境智能体：负责作为环境 遵守环境规律和角色互动并更改环境变量 触发环境事件
- 总结智能体：在最后按要求生成文本（比如网文、剧本）
- 氛围智能体：一个轻量化智能体 用于快速决定一些变量是否要被注入上下文（当然也可以由关键词检测替代）

### 具体流程
1. 给定非结构化种子文本（组），通过GraphRAG建立知识/关系图谱
2. 生成类似SillyTavern的角色设定给角色智能体 生成对应的规则等给环境智能体 环境变量生成等 生成剧情目标给导演智能体 此时暂停给允许人工修改
3. 导演智能体开始工作，通过AutoGen来作为场景，让指定Agent设定好的环境下进行一定轮次的自由互动，行动前先自动快照
4. 导演智能体在每次互动完后，导演智能体重新接手（可以看具体内容、梗概、角色记忆、环境情况等等）判断是否要修改场景条件然后重拍、额外轮次的模拟、进入下一个场景
5. 进入下一个场景时，角色的上一轮记忆会被处理并放入数据库变成长期记忆
6. 当导演智能体认为可以结局时 角色智能体的各种动作、发言等会被导出到一个文件 然后总结智能体会开始按照一开始的用户要求生成流畅的文本。（总结智能体可以概览所有场景发生的具体情况或概要、查看具体智能体的记忆或进行采访，以便加深理解）

### 技术说明
- 角色智能体会有系统提示词、环境信息、角色设定（静态的 还有些会根据关键词或者一个特别的“氛围Agent”决定额外加入来强调）、长期记忆（跨轮次已压缩的记忆）、短期记忆（轮次内对话）等 适合的编排来提高注意力并在保持质量的情况下增加能缓存的部分
- 不同的智能体（以及不同分级的角色）可以使用相同或不同的LLM配置（不同的模型ID甚至接入点）来优化成本
- 最后要保留分支能力，用户可以考虑从指定快照开始分支推演（有点像Git）
